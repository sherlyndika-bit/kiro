data RAB/Fee di Google Sheet, dan (c) alur handoff AI ↔ manusia.

## Bug yang diperbaiki di PR ini

### 1. ❌→✅ `/toggle-mode` tidak ada handler-nya
Dashboard memanggil `/toggle-mode` saat operator klik **Ambil Alih / Kembali ke AI**,
tapi **tidak ada workflow** yang menerima path itu (WF5 ping malah melaporkan
`toggle_mode: active` — bohong). **Fix:** tambah `workflow-6-toggle-mode.json`
yang meng-update `conversations.mode` di Supabase.

### 2. ❌→✅ AI tetap balas walau sudah di-Take Over (handoff bocor)
WF1 **tidak pernah mengecek `mode`** percakapan, jadi setelah operator ambil alih
(mode=manual), AI **tetap auto-reply** — bertabrakan dengan operator. **Fix:**
- WF1 incoming upsert **tidak lagi menimpa kolom `mode`** (dulu paksa `"ai"`,
  yang bahkan menghapus status manual tiap ada pesan masuk).
- Tambah node **"AI Active? (Mode Gate)"** sesudah Build Decision: cek
  `conversations.mode`. Kalau `manual` → AI berhenti (return []). Kalau `ai` →
  lanjут kirim. Fail-open (kalau cek gagal, tetap kirim) supaya customer tidak ghosting.

### 3. ❌→✅ Harga RAB/Fee di WF2 tidak sama dengan Google Sheet
Tabel harga di WF2 berbeda dari sheet (mis. Rumah Standar 5–6jt vs sheet 4,5–6jt;
Villa Premium 12–16jt vs sheet 10–12jt) dan fee pakai range %/kategori yang salah.
**Fix:** WF2 di-rewrite agar **identik dengan sheet** (RAB per tipe+tier) dan fee
deterministik per tier: Standar 7%, Menengah 5%, Premium 4% — sama dengan
`pricingService.ts` di dashboard.

### 4. ❌→✅ Node WhatsApp send default ke `sendTemplate`
WF2 (estimasi), WF3 (notif proposal + follow-up), WF0 (dashboard message) belum
punya `operation: send` + `messageType: text` eksplisit, sehingga free-form text
gagal (butuh template Meta yang di-approve). **Fix:** semua node WA sekarang
eksplisit `resource=message, operation=send, messageType=text, previewUrl=false`
(sama seperti perbaikan WF1 di PR #11).

## Catatan / belum diubah (sengaja)
- **WF3 masih HTML "kertas"**, belum pakai template proposal visual (deck + foto)
  yang dipakai dashboard. Untuk dipersamakan, port `proposalTemplate.ts` ke node
  Build HTML WF3 + storage publik untuk PDF (langkah berikutnya).
- **Anon key Supabase ter-hardcode** di workflow. Untuk pemakaian pribadi OK;
  untuk produksi sebaiknya pindah ke n8n Credentials (Header Auth) atau Variables.
- **WF2/WF3 belum baca Google Sheet langsung** — angka disinkronkan manual.
  Kalau mau live, ganti node "Calculate" dengan Google Sheets read + lookup.
- File lama `workflow-1-incoming-message.json` (v1) & `-v2` dibiarkan sebagai
  arsip. **Pakai v3** (memory + mode gate).
